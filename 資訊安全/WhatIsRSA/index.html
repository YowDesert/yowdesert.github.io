Topic : RSA 是什麼？

RSA 是一種 非對稱加密演算法

一、什麼是非對稱加密？

  • 非對稱加密使用「一對金鑰」：
    • 🔓 公開金鑰（Public Key）：可以提供給他人用來加密資料
    • 🔐 私密金鑰（Private Key）：僅自己持有，用於解密資料
  • 相較於對稱加密（只有一把金鑰），非對稱加密像 RSA 可以：

    ✅ 讓「別人加密給你」，但只有你能解開

二、RSA 與對稱加密比較

比較項目 對稱加密 非對稱加密（RSA）
🔑 金鑰數量 1 把 2 把（公+私）
⚡ 加解密速度 快 慢
📦 加密大量資料 適合 不適合
🔁 金鑰交換難度 高（要先傳） 低（公鑰可公開）
🔏 支援簽章驗證 否 是
🔧 常見用途 檔案加密、VPN 傳輸金鑰、身份驗證、簽章

三、RSA 的核心數學概念

RSA 建立在一個重要的數學困難上：

大數質因數分解很難

  1. 選兩個大質數：p 和 q
  2. 計算 $N = p × q$（模數）
  3. 計算歐拉函數：$φ(N) = (p-1)(q-1)$
  4. 選一個加密指數 e（常見為 65537）
  5. 計算私鑰 d，使得：$e × d ≡ (1)mod (N)$（d 是 e 的模反元素）
  6. 結果：
  • 公開金鑰：(e, N)
  • 私密金鑰：(d, N)

四、RSA 加解密流程

  • 🔐 加密（使用公開金鑰）：
    假設你想加密明文 m（要先轉成整數），那就算：
    $ c = m^e mod N$
    m 是明文（轉成整數後）

  • 🔓 解密（使用私密金鑰）：
    $ m = c^d mod N$

✅ 只有擁有私鑰的那個人能成功解密！

五、應用場景

  • ✅ 加密敏感資料（像 HTTPS 傳輸）
    -✅ 數位簽章（驗證訊息是否真的是某人發的）
  • ✅ 金鑰交換（混合式加密中，先用 RSA 傳送對稱加密金鑰）

六、RSA 的安全性來自?

  • 它的安全性來自於：
    目前沒有有效的方式在合理時間內把一個很大的 N 拆解為 p 和 q。
    如果有人能快速分解大數，RSA 就不安全了（例如量子電腦未來可能威脅 RSA）。

總結

項目 說明
金鑰種類 非對稱（公開+私密）
安全基礎 大數質因數分解困難
常用用途 加密、簽章、金鑰交換
常見公開指數 65537
